# §5.9 cross-platform RAG E2E — bootstrap CIs (B=10000, seed=42)

Generated by `research/scripts/bootstrap_rag_cross_platform.py`.
Each row is one Android device that ran the full 2×2 ablation
(BGE-small embedder + Qwen 2.5 0.5B / 1.5B Q4_K_M, 200 NQ queries).
Kirin 659 (the only ARMv8.0 / Cortex-A53 chip in the table) runs
the bench with `Algorithm.FLAT` instead of `Algorithm.HNSW`. The
eight-pass diagnostic that root-caused this and shipped the
workaround is documented in
`research/results/cross_platform_e2e/ane_lx3_kirin659_investigation.md`.
FLAT is exact (brute-force scan), HNSW is approximate with > 99 %
recall at this scale, so per-cell `F1_short` differs by at most
~0.005 between the two algorithms — within the bootstrap CI.
