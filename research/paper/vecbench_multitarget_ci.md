# Multi-target build — Round 1 (dispatched/auto) vs Round 2 (baseline-forced apples-to-apples)

Bootstrap method: percentile, paired-query resampling for ratios. `B = 10000`,
seed = 42. The two rounds run the **exact same** harness (`vector-bench-paper384-scale`),
the same 100-query set, on the same 4 physical devices. The only difference is
which native library `dev.dazzle.sdk.DazzleNativeLoader` resolves at process
init: round 1 follows `/proc/cpuinfo` and chooses the variant the chip can run
at hardware speed, round 2 forces every chip onto `libdazzle.so` baseline
(armv8-a + -mcpu=generic).
